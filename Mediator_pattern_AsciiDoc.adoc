### * Mediator Pattern :
Mediator design pattern (aracý tasarým deseni), behavior grubununa ait, çalýþmalarý birbirleri ile ayný arayüzden türeyen nesnelerin durumlarýna baðlý olan nesnelerin davranýþlarýný düzenleyen tasarým desenidir.

Mediator tasarým deseni birbirleri ile iliþkili olan bu nesneler arasýnda ki iletiþimin ana bir nesne üzerinden (mediator) yapýlmasýný saðlar. *  Mediator: Nesneler arasýndaki iliþkiyi saðlayacak metotlarýn tanýmlý olduðu arayüz.

*  ConcreteMediator: Nesneler arasýndaki iliþkiyi saðlayacak gerçek nesnedir. Mediator arayüzünü uygular. Ýçinde Colleague ara yüzünden türeyen nesnelerin listesini barýndýrýr.

*  ConcreteColleague: ConcreteMediator üzerinden birbirleri ile iliþkili nesnelerdir. Colleague arayüzünü uygularlar.

public class BankaSubesi_Impl extends BankaSubesi{

	public BankaSubesi_Impl(MerkeziBankaMediator med, String mesaj){
		super(med,mesaj);
	}
	@Override
	public void sendMesaj(String mesaj) {
		
		System.out.println(this.subeIsmi + " Tarafýndan bu mesaj gönderildi : "+mesaj );

	}

package Mediator_Pattern;

import java.util.ArrayList;
import java.util.List;

public class MerkeziBankaMediator_Impl implements MerkeziBankaMediator{

	private List<BankaSubesi> subeList;
	
	public MerkeziBankaMediator_Impl(){
		this.subeList = new ArrayList<>();
	}
	@Override
	public void sendBilgilendirmeMesaj(String mesaj, BankaSubesi sube) {
		
		for (BankaSubesi s : this.subeList){
			if (s != sube){
				s.receiveMesaj(mesaj);
			}
		}
		
	}

	@Override
	public void addSube(BankaSubesi sube) {
		this.subeList.add(sube);
	}

}
